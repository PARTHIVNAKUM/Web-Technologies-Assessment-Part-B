User Security Concepts

Analysis of security flaws and best practices as covered in Unit 9.

Common Registration Flaws

  • Storing passwords in plain text.
  • Weak password requirements (e.g. allowing "123").
  • Lack of input sanitization (leading to XSS/SQLi).
  • No privacy policy consent.

Admin Privileges

Admin accounts have elevated access to:

  • Manage Users (Create, Delete, Ban).
  • Moderate Content.
  • Configure System Settings.

Secure Form Design Principles

When designing forms, always ensure:

  • Validation: Check data types and formats on both client and server (see Contact page).
  • Encryption: Hash passwords using strong algorithms (bcrypt/Argon2) before storage.
  • HTTPS: Serve all pages over SSL/TLS.
  • CSRF Protection: Use anti-CSRF tokens for state-changing requests.
